# paper-refiner tool 小白配置指南

面向**第一次使用**本项目的同学：从零到能在浏览器里打开网页、上传论文并调用大模型改写。按顺序做即可。

---

## 一、这个工具是干什么的？

- 上传 **Word 文档（`.docx`）**。
- 程序会列出每一段，标出**能不能改**（表格、公式、图里的段落一般会跳过）。
- 你勾选若干段，选一个**提示词（Prompt）**，点「生成改写」，程序会调用大模型 API，**一段一段**改写。
- 你在网页里检查、可手动改字，再**导出**成新的 `.docx`。

> **重要**：请遵守学校对论文原创性与 AI 使用的规定。本工具不保证任何查重 / AIGC 检测结果。

---

## 二、开始前你需要准备什么？

| 项目 | 说明 |
|------|------|
| **电脑或云服务器** | Windows / macOS / Linux 均可；能上网。 |
| **Python** | **3.10 或以上**（3.12 也可以）。终端里输入 `python --version` 或 `python3 --version` 能看到版本号即可。 |
| **大模型 API** | 任选其一：**DeepSeek**、**gptsapi**、**OpenAI 官方**等，只要提供 **OpenAI 兼容** 的 `https://.../v1/chat/completions` 即可。下面以 **DeepSeek** 为例。 |
| **论文文件** | 必须是 **`.docx`**。老版 `.doc` 请先在 Word 里「另存为」→ `.docx`。 |

## 三、把项目放到本机

如果你是用 **Git** 克隆的：

```bash
git clone <你的仓库地址>
cd paper-refiner-tool
```

如果是 **压缩包** 解压的，用终端进入解压后的文件夹，确保能看到 `pyproject.toml`、`prompts.yaml` 等文件。

**之后所有命令尽量都在这个文件夹里执行**（或在任意目录执行，但 `.env` 必须放在**项目根目录**，见下文）。

## 四、安装 Python 依赖（只做一次）

### 4.1（推荐）用虚拟环境，避免和系统 Python 混在一起

```bash
cd paper-refiner-tool
python3 -m venv .venv
```

**激活虚拟环境：**

- Linux / macOS：
  ```bash
  source .venv/bin/activate
  ```
- Windows（CMD）：
  ```cmd
  .venv\Scripts\activate.bat
  ```
- Windows（PowerShell）：
  ```powershell
  .venv\Scripts\Activate.ps1
  ```

激活成功后，命令行前面一般会多 `(.venv)` 字样。

### 4.2 安装本项目

```bash
pip install -U pip
pip install -e .
```

没有报错、最后出现 `Successfully installed paper-refiner-tool` 即表示安装成功。

## 五、配置 API（密钥 + 地址 + 模型）

### 5.1 复制示例配置文件

在项目根目录（和 `prompts.yaml` 同一层）执行：

```bash
cp .env.example .env
```

然后用**记事本 / VS Code / Cursor** 打开 `.env` 编辑。

### 5.2 填写 DeepSeek（示例）

`.env` 里与 API 相关的几项含义：

| 变量名 | 写什么 |
|--------|--------|
| `OPENAI_API_BASE` | DeepSeek 填：`https://api.deepseek.com`（**不要**随便加路径，程序会自动补 `/v1`） |
| `OPENAI_API_KEY` | 在 [DeepSeek 开放平台](https://platform.deepseek.com/api_keys) 创建的 **API Key**，一般以 `sk-` 开头 |
| `OPENAI_MODEL` | 例如：`deepseek-v4-flash` 或 `deepseek-v4-pro`（以平台当前文档为准） |

**注意：**

- 等号两边**不要多空格**。
- 密钥**整行粘贴**，一般**不要加英文引号**。
- `.env` 已在 `.gitignore` 里，**不要**把含真实密钥的 `.env` 上传到 GitHub。

### 5.3 可选：加快批量改写（多段同时请求）

默认会同时发 **5** 个请求（可减轻「一百多段很慢」的问题）。若经常被限流（429），可调小；若网络稳定可调大：

```bash
OPENAI_MAX_CONCURRENT=5
```

## 六、启动网页版（最常用）

### 6.1 在项目根目录启动

确保已激活虚拟环境（若用了 venv），然后：

```bash
cd paper-refiner-tool
paper-refiner-web
```

看到类似 `Uvicorn running on http://0.0.0.0:8765` 就表示启动成功。

### 6.2 用浏览器打开

本机浏览器地址栏输入：

**http://127.0.0.1:8765**

（若在远程服务器 / AutoDL 上跑，请用平台提供的「自定义服务 / 端口映射」等方式访问对应端口。）

### 6.3 不想把密钥写进 `.env` 时

也可以每次启动时写在命令行（**不要**把含密钥的命令截图发给别人）：

```bash
paper-refiner-web \
  --api-key sk-你的密钥 \
  --api-base https://api.deepseek.com \
  --model deepseek-v4-flash
```

命令行传入的变量**优先级高于** `.env`。

### 6.4 端口被占用（`address already in use`）

说明 **8765 已经被别的程序占用**（常见是上一次 `paper-refiner-web` 没关）。

- **做法 A**：关掉旧进程后再启动  
  ```bash
  pgrep -af paper-refiner-web
  kill <上面显示的 PID>
  ```
  若没有 `pgrep`，可换端口：  
  ```bash
  paper-refiner-web --port 8766
  ```
  然后浏览器打开 **http://127.0.0.1:8766**

- **做法 B**：程序已会在终端打印中文提示，按提示操作即可。

## 七、网页里怎么用？（四步）

1. **上传** `.docx`。
2. **勾选**要改写的段落（可「全选可改写」；右侧会显示已选数量 / 单次上限）。
3. 选好 **Prompt**、温度等，点 **「生成改写」**（多段会并行请求 API，仍可能需几分钟，请耐心等待）。
4. 在结果里检查、可编辑，勾选 **「采纳本段」**，点 **「导出已采纳段落为 docx」** 下载。

刷新页面后，若之前上传过同一文档，可能会**自动恢复上次会话**（勾选与设置会保存）；要换论文可点 **「重新上传」**。

## 八、常见问题（FAQ）

### Q1：提示 401 / Token invalid？

密钥错、过期、或复制不全。到对应平台重新复制 **完整** API Key，改 `.env` 或命令行参数后**重启** `paper-refiner-web`。

### Q2：提示 429 / 限流？

并发太高或短时间请求太多。把 `.env` 里的 `OPENAI_MAX_CONCURRENT` 改小（例如 `3`），或一次少选几段，重启服务后再试。

### Q3：连接超时、很慢？

网络到 API 不稳定。可设置重试（见 `.env.example` 里 `OPENAI_RETRY_COUNT` 等说明），或换时段、换网络环境。

### Q4：改完格式和 Word 里不一样？

当前实现是**整段纯文本替换**，段内原来的加粗、多种字体可能被**压成一种**；这是已知限制。重要版式请自己在 Word 里再调。

### Q5：想换提示词 / 自己写 Prompt？

编辑项目根目录的 **`prompts.yaml`**：每个模板里必须有且仅有一个 **`{text}`** 占位符。改完后**刷新网页**即可在下拉框里看到（无需重装 pip，除非改了 Python 代码）。

### Q6：单次最多能选多少段？

默认 **1000** 段（环境变量 `PAPER_REFINER_MAX_BATCH` 可调）。段数越多，总耗时越长。

## 九、命令行（可选）

适合习惯用终端的同学：

```bash
paper-refiner --help
paper-refiner prompts
paper-refiner list-doc 你的论文.docx
```

改写示例（索引需自己从 `list-doc` 里看）：

```bash
paper-refiner rewrite 你的论文.docx --indices 10,11,12 --prompt-id vip_cn_zero --output 输出.docx
```
